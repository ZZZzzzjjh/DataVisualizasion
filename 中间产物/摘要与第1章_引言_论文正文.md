# 摘要

随着高校毕业生就业竞争加剧，选择在哪个城市就业和生活成为许多毕业生需要面对的重要问题。不同城市在就业机会、薪资水平、生活成本和住房租金方面存在明显差异。部分城市虽然工资较高，但生活成本和房租也较高，实际可支配收入不一定理想；部分新一线或二线城市虽然薪资略低，但租金和生活成本相对可控，扣除主要支出后的月结余反而更稳定。基于这一现实问题，本文以北京、上海、广州、深圳、杭州、南京、苏州、成都、重庆、武汉、西安、长沙、郑州、天津、青岛、宁波、厦门、合肥、佛山和东莞等 20 个城市为研究对象，综合整理城市经济人口数据、生活成本与租金数据、就业薪资与岗位结构数据，构建城市就业与生活成本综合分析数据集。

本文首先对多源数据进行采集、清洗、单位统一和指标构建，形成可用于分析的城市综合数据。随后通过描述性统计和可视化方法，比较不同城市的 GDP、人口、税后月薪、生活成本、租金和实际月结余等指标，并结合行业结构和岗位类型分析城市就业市场差异。在此基础上，本文进一步进行城市类型划分，构建城市推荐分模型，从月结余、平均薪资、租金、生活成本和就业热度等角度对城市进行综合排序。最后，本文基于 Streamlit 开发交互式 Web 应用，实现城市总览、单城市画像、多城市对比、就业结构分析、成本收支分析、薪资匹配分析、城市分型、推荐助手和 DeepSeek 智能分析等功能。

研究结果表明，城市就业选择不能只看 GDP 和平均工资，还需要综合考虑生活成本、租金压力、岗位结构和实际月结余。上海、深圳、北京等城市薪资水平较高，但房租和生活成本也较高；长沙、成都、南京、宁波、青岛等城市在扣除生活成本和房租后的实际月结余方面表现较好。本文研究能够为毕业生就业城市选择提供一定参考，也为城市就业与生活成本的可视化分析提供了一个课程项目实践案例。

关键词：城市就业；生活成本；薪资匹配；城市推荐；数据可视化；Streamlit

# 1 引言

## 1.1 课题背景与意义

近年来，高校毕业生人数持续增加，毕业生在求职时不仅需要选择行业和岗位，也需要选择未来工作和生活的城市。对于很多毕业生来说，城市选择会直接影响就业机会、薪资水平、生活成本、住房压力以及未来发展空间。因此，“去哪个城市工作更合适”已经不只是一个生活偏好问题，也逐渐成为一个需要结合数据进行分析的问题。

在现实生活中，很多人在判断城市是否适合就业时，容易只关注工资水平。例如，北京、上海、深圳、杭州等城市往往给人“机会多、工资高”的印象，但这些城市的房租和日常消费也比较高。如果只看工资，不考虑生活成本和租金，就可能高估这些城市的实际吸引力。相反，一些新一线或二线城市虽然平均薪资低于一线城市，但由于租金较低、生活成本较可控，扣除基本生活支出和房租后，实际剩余收入可能更高。

因此，本文选择“城市就业与生活成本分析”作为研究主题，试图从数据角度比较不同城市之间的就业收入和生活压力。本文不仅关注城市平均薪资，也关注单人月生活成本、市中心一居室租金、扣除生活成本和房租后的实际月结余，以及不同行业和岗位结构。这样可以更全面地分析城市就业生活性价比，而不是简单地用工资高低判断城市好坏。

本课题具有一定现实意义。对于毕业生而言，本文可以帮助其更直观地比较不同城市的就业和生活压力，为选择就业城市提供参考；对于课程学习而言，本文将数据采集、数据清洗、可视化分析、建模评价和 Web 应用开发结合起来，能够体现数据可视化课程中从数据到系统展示的完整流程；对于城市比较分析而言，本文也提供了一种将就业机会、薪资水平、生活成本和租金压力结合起来的分析思路。

## 1.2 国内外研究现状简述

关于城市就业和生活成本的研究，国内外已有较多相关成果。国外研究中，城市吸引力、就业机会、生活成本和居民收入水平经常被放在一起讨论。一些研究会通过收入、住房支出、消费价格和公共服务等指标分析城市生活质量，也有研究关注不同城市之间的人才流动和就业机会差异。这类研究通常强调，城市竞争力不仅取决于经济规模，也取决于城市能否为居民提供相对合理的生活成本和发展机会。

国内相关研究主要集中在城市经济发展、城市吸引力评价、就业质量评价和居民生活成本分析等方面。例如，一些研究会使用 GDP、人口、居民收入、产业结构和公共服务水平等指标评价城市综合发展水平；也有研究关注高校毕业生就业流向，分析一线城市、新一线城市和二线城市对毕业生的吸引力差异。近年来，随着房租和生活成本问题受到关注，越来越多研究开始强调不能只用工资水平衡量城市就业吸引力，还需要考虑住房压力和实际可支配收入。

在数据可视化方面，国内外也有较多实践案例。许多城市研究项目会使用柱状图、散点图、地图、热力图和排名图展示城市差异。相比传统表格，可视化图表能够更直观地表现城市之间的差距，也更适合普通用户理解。近年来，Streamlit、Dash 等工具降低了数据分析系统的开发门槛，使课程项目和小型研究也可以快速搭建交互式数据应用。

不过，从已有研究和实践来看，仍然存在一些不足。第一，部分城市比较只关注经济规模或平均工资，对生活成本和租金压力考虑不够。第二，部分生活成本分析只展示消费指数，缺少更直观的人民币月度金额。第三，一些数据分析停留在静态图表层面，用户无法根据自己的需求筛选城市或调整条件。基于这些问题，本文尝试将城市就业、薪资、生活成本、租金和岗位结构整合到同一分析框架中，并开发交互式 Web 应用，使分析结果更直观、更易使用。

## 1.3 本课题的研究内容与创新点

本文的研究内容主要包括数据采集与预处理、城市就业与生活成本描述性统计分析、城市分型与推荐模型构建，以及 Streamlit 交互式 Web 应用开发四个方面。

首先，在数据采集与预处理方面，本文选取 20 个代表性城市作为研究对象，综合整理城市基础信息、GDP、人口、生活成本、租金、税后月薪和就业结构等数据。由于数据来源较多，本文对城市名称、字段格式、数值单位和缺失值进行了统一处理，并构建了人均 GDP、扣除生活成本后的月结余、扣除生活成本和房租后的实际月结余等衍生指标。

其次，在描述性统计与可视化分析方面，本文从城市宏观发展背景、就业市场结构、生活成本水平和收入成本匹配关系四个角度展开分析。通过 GDP 对比、人口对比、薪资排名、生活成本对比、租金对比、实际月结余排名和四象限图等方式，比较不同城市之间的差异。

再次，在数据建模与深度洞察方面，本文根据薪资、生活成本、租金和实际月结余对城市进行类型划分，并构建城市推荐分模型。推荐模型综合考虑月结余、平均薪资、租金、生活成本和就业热度代理指标，使城市排序不再只由工资水平决定，而是更强调收入和成本之间的匹配关系。

最后，在系统开发方面，本文基于 Streamlit 构建交互式 Web 应用。系统支持城市筛选、单城市画像、多城市对比、就业结构可视化、成本收支分析、薪资匹配分析、城市分型、推荐助手、AI 助手和数据管理等功能。系统通过 Plotly 绘制交互式图表，并接入 DeepSeek API 生成智能分析和推荐解释。

本文的创新点主要体现在以下三个方面。第一，本文将就业收入、生活成本和租金压力放在同一分析框架中，强调实际月结余对城市选择的参考意义。第二，本文尽量使用“元/月”口径展示薪资、生活成本和租金，使分析结果比抽象指数更容易理解。第三，本文不仅完成静态论文分析，还开发了交互式 Web 应用，使用户可以根据区域、城市等级和城市选择进行动态比较，并通过推荐助手获得更直观的城市选择建议。

总体而言，本文希望通过数据分析和可视化系统，帮助毕业生更全面地理解不同城市的就业机会和生活成本差异，为就业城市选择提供参考。
