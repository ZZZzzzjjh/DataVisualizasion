# 数据清洗说明

本文档说明《城市就业与生活成本分析》项目的数据清洗方法、处理流程、代码运行方式和清洗后数据文件。

## 1. 清洗目标

本项目的数据来自多个公开来源，包括 Numbeo、Wikipedia 城市 GDP 与人口表、DataV 地图边界以及手工整理的核心城市清单。不同来源之间存在字段命名不统一、脚注符号、数值格式不一致、城市名称中英文混用、部分指标缺失等问题。

数据清洗的目标如下：

- 统一城市名称字段，建立可合并的城市维度。
- 去除网页表格中的脚注、特殊符号和无关标记。
- 将 GDP、人口、生活成本指数等字段转换为数值型。
- 删除重复记录，保留每个城市一条核心记录。
- 合并城市基础信息、GDP、人口和生活成本数据。
- 生成可直接用于可视化分析的城市综合面板表。
- 输出数据质量报告，记录缺失值情况。

## 2. 原始数据文件

## 3. 数据清洗流程

### 3.1 字段命名统一

不同数据表中的城市字段命名不一致，例如：

- `city_zh`
- `city_en`
- `cities cities`
- `城市 城市`
- `City`

清洗时统一为：

| 标准字段 | 含义 |
| --- | --- |
| `city_name_zh` | 城市中文名 |
| `city_name_en` | 城市英文名 |
| `province_zh` | 中文省份 |
| `province_en` | 英文省份 |

统一字段名之后，可以使用 `city_name_en` 作为主键合并英文来源表格，用 `city_name_zh` 作为中文展示字段。

### 3.2 脚注与特殊符号清理

网页表格中常见脚注和特殊符号，例如：

```text
Shanghai[7][8]
Beijing⍟#
Shenzhen†
```

这些符号会影响城市名称匹配，因此清洗时使用正则表达式去除：

```python
text = re.sub(r"\[[^\]]*\]", "", text)
text = re.sub(r"[#†‡*~⍟]", "", text)
```

处理后结果：

```text
Shanghai[7][8] -> Shanghai
Beijing⍟# -> Beijing
Shenzhen† -> Shenzhen
```

### 3.3 数值字段转换

部分数值字段中带有逗号、百分号、加号或其他文本，例如：

```text
5,392,671
+13.51%
40.5
```

清洗时统一去除无关符号，并转换为数值型：

```python
text = text.replace(",", "").replace("%", "").replace("+", "").strip()
match = re.search(r"-?\d+(?:\.\d+)?", text)
```

主要转换字段包括：

| 字段 | 处理方式 |
| --- | --- |
| `gdp_2024_cny_million` | 转换为数值，单位为百万元人民币 |
| `gdp_2024_usd_million` | 转换为数值，单位为百万美元 |
| `population_2020` | 转换为数值，单位为人 |
| `population_2010` | 转换为数值，单位为人 |
| `population_change_percent` | 转换为百分比数值 |
| `cost_of_living_index` | 转换为数值 |
| `rent_index` | 转换为数值 |
| `local_purchasing_power_index` | 转换为数值 |

### 3.4 重复值处理

在 GDP 表和人口表中，理论上每个城市只应保留一条记录。清洗时按照城市英文名去重：

```python
df = df.drop_duplicates(subset=["city_name_en"])
```

如果后续发现同一城市存在多个统计口径，应优先保留与课程分析口径一致的数据，并在数据说明中注明。

### 3.5 多表合并

本项目以 `city_basic.csv` 中的 20 个核心城市为基础表，依次合并 GDP、人口和生活成本数据。

合并方式：

```python
panel = basic.merge(gdp, on="city_name_en", how="left")
panel = panel.merge(population, on="city_name_en", how="left")
panel = panel.merge(cost, on="city_name_en", how="left")
```

使用 `left join` 的原因是：保证 20 个核心城市全部保留，即使某些城市暂时缺少 Numbeo 生活成本数据，也不会被删除。

### 3.6 衍生指标计算

清洗后额外计算了一个人均 GDP 指标：

```python
gdp_per_capita_2024_cny = gdp_2024_cny_million * 1_000_000 / population_2020
```

该指标可以用于比较不同城市的经济发展水平。

同时，为后续综合评价做准备，对部分指标进行了 0-1 标准化。

正向指标标准化：

```text
标准化值 = (原始值 - 最小值) / (最大值 - 最小值)
```

反向指标标准化：

```text
反向标准化值 = 1 - (原始值 - 最小值) / (最大值 - 最小值)
```

其中：

| 指标 | 方向 | 说明 |
| --- | --- | --- |
| `gdp_index` | 正向 | GDP 越高，经济规模越大 |
| `population_index` | 正向 | 人口规模越大，城市规模越大 |
| `living_cost_index_norm` | 反向 | 生活成本越低，得分越高 |
| `rent_index_norm` | 反向 | 租金成本越低，得分越高 |
| `purchasing_power_index_norm` | 正向 | 本地购买力越高，得分越高 |

### 3.7 参考性综合指数

由于目前还没有完整的就业岗位和薪资数据，因此清洗脚本暂时生成了一个参考性指标：

```text
city_living_value_reference_index
```

计算方式：

```text
城市生活价值参考指数 =
平均值(gdp_index, living_cost_index_norm, rent_index_norm, purchasing_power_index_norm)
```

该指标只能作为阶段性参考，后续获得就业数据后，建议替换为正式的“就业生活性价比指数”：

```text
就业生活性价比指数 =
标准化岗位数量 + 标准化平均薪资 - 标准化生活成本
```

### 3.8 缺失值处理

缺失值不直接删除，而是保留并统计。原因是：

- GDP 和人口数据对 20 个核心城市匹配较完整。
- Numbeo 只覆盖部分中国城市，如果直接删除缺失值，会导致城市样本减少。
- 可视化时可以根据分析目标选择是否过滤缺失城市。

清洗脚本生成 `missing_key_fields` 字段，表示每个城市关键字段缺失数量。

同时生成 `data_quality_report.csv`，统计每个字段的：

- 缺失数量
- 缺失率
- 唯一值数量

## 6. 核心代码说明

### 6.1 去除脚注符号

```python
def remove_footnotes(value: object) -> str:
    if pd.isna(value):
        return ""
    text = str(value)
    text = re.sub(r"\[[^\]]*\]", "", text)
    text = re.sub(r"[#†‡*~⍟]", "", text)
    return text.strip()
```

### 6.2 数值转换

```python
def to_number(value: object) -> float | None:
    if pd.isna(value):
        return None
    text = remove_footnotes(value)
    text = text.replace(",", "").replace("%", "").replace("+", "").strip()
    match = re.search(r"-?\d+(?:\.\d+)?", text)
    if not match:
        return None
    return float(match.group())
```

### 6.3 0-1 标准化

```python
def min_max(series: pd.Series, reverse: bool = False) -> pd.Series:
    values = pd.to_numeric(series, errors="coerce")
    min_value = values.min()
    max_value = values.max()
    if pd.isna(min_value) or pd.isna(max_value) or min_value == max_value:
        result = pd.Series([None] * len(values), index=values.index)
    else:
        result = (values - min_value) / (max_value - min_value)
    if reverse:
        result = 1 - result
    return result.round(4)
```
