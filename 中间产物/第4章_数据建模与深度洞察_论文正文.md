# 4 数据建模与深度洞察

前一章主要通过描述性统计和图表对比展示了不同城市在薪资、生活成本、租金和月结余方面的差异。本章在此基础上进一步进行城市类型划分和推荐模型构建。本文的建模目的不是追求复杂算法，而是希望用比较直观的指标，把城市就业生活差异转化为更容易理解的城市类型和推荐结果，为毕业生选择就业城市提供参考。

如图 4-1 所示，本章的分析流程先从薪资、生活成本、租金、就业热度等指标出发，对城市进行初步分型；然后结合行业和岗位结构分析不同城市适合的人群；最后构建城市推荐分，对城市进行排序并解释推荐结果。

## 4.1 城市类型划分

城市类型划分的目的是将多个指标综合起来，识别不同城市的就业生活特征。如果只看税后月薪，容易把北京、上海、深圳等高薪城市排在最前；但如果加入房租和生活成本，就会发现部分高薪城市实际月结余并不高。因此，本文主要依据平均薪资、生活成本、租金和实际月结余等指标，对样本城市进行分型。

### 4.1.1 基于薪资、生活成本和租金的城市分型

本文将城市大致划分为高薪高成本型、均衡发展型、低成本友好型、就业机会突出但成本压力型和收入成本压力型五类。具体划分思路如表 4-1 所示。

<a id="tab4-1"></a>

表 4-1 城市类型划分规则

| 城市类型 | 主要判断依据 | 代表城市 |
|---|---|---|
| 高薪高成本型 | 税后月薪较高，但生活成本和租金也较高，实际月结余不一定高 | 北京、上海、深圳、广州 |
| 均衡发展型 | 薪资水平较好，生活成本和租金压力相对可控，月结余表现较稳定 | 南京、成都、宁波、杭州、苏州 |
| 低成本友好型 | 薪资不一定最高，但租金较低，扣除成本后月结余较好 | 长沙、青岛、天津、郑州、东莞 |
| 就业机会突出但成本压力型 | 城市规模和就业热度较高，但实际月结余表现一般 | 武汉、重庆 |
| 收入成本压力型 | 薪资或月结余偏低，需要结合具体岗位谨慎判断 | 西安、厦门、合肥、佛山 |

如表 4-1 所示，高薪城市并不一定都是最适合毕业生长期生活的城市。以北京和上海为例，这类城市就业机会多、薪资水平高，但房租和生活成本也明显偏高，因此实际月结余并不一定理想。相反，长沙、青岛、天津等城市虽然薪资不属于最高，但租金压力较小，扣除生活成本和房租后仍能保留一定结余。

这种分型方式的优点是比较直观，能够把复杂指标转化为容易理解的城市标签。对于毕业生来说，城市标签比单独的数字排名更容易辅助决策。例如，追求高薪和平台机会的学生可以关注高薪高成本型城市，而预算敏感、希望生活压力小一些的学生可以优先考虑低成本友好型或均衡发展型城市。

### 4.1.2 基于就业结构和岗位特征的城市对比

仅根据薪资和成本分型还不够，因为不同城市的行业结构和岗位类型也会影响毕业生选择。本文进一步结合就业结构数据，对不同城市的主要行业和岗位特征进行比较，结果如表 4-2 所示。

<a id="tab4-2"></a>

表 4-2 部分城市就业结构特征

| 城市 | 主要行业方向 | 代表岗位特征 |
|---|---|---|
| 北京 | 互联网/软件、金融/商务服务、科研教育 | 算法、数据分析、金融分析、研究助理 |
| 上海 | 金融/商务服务、互联网/软件、贸易物流 | 金融分析、产品运营、跨境贸易、供应链 |
| 深圳 | 互联网/软件、电子信息、先进制造 | 软件开发、硬件工程、嵌入式、金融风控 |
| 杭州 | 互联网/软件、电子商务、金融科技 | 产品运营、电商运营、数据分析 |
| 南京 | 先进制造、互联网/软件、贸易物流 | 工艺工程、质量工程、数据分析、供应链 |
| 长沙 | 先进制造、互联网/软件、教育培训 | 设备工程、开发、课程运营 |
| 东莞 | 先进制造、电子信息、贸易物流 | 生产质量、硬件测试、外贸供应链 |

如表 4-2 所示，不同城市不仅薪资和生活成本不同，就业机会结构也不同。北京、上海、深圳和杭州更偏向互联网、金融、数据和电子信息方向；南京、长沙、东莞等城市则在先进制造、供应链和工程类岗位方面更明显。因此，城市选择还需要结合个人专业方向。如果学生学习计算机、数据分析、金融等方向，可以更多关注北京、上海、深圳和杭州；如果学生关注制造业、供应链、工程技术等方向，苏州、南京、东莞、佛山等城市也具有一定吸引力。

## 4.2 城市推荐模型构建

城市推荐模型的目标是将多个指标合并为一个综合参考分，用于辅助城市排序。需要说明的是，推荐分并不是绝对标准，也不是让用户只看一个分数做决定，而是作为后台排序方法。系统前端仍然主要展示税后月薪、生活成本、租金和月结余等直观指标。

### 4.2.1 推荐指标选择

本文推荐模型选择了五个主要指标：扣除生活成本和房租后的月结余、平均税后月薪、市中心一居室租金、单人月生活成本和就业热度代理指标。各指标的方向和权重如表 4-3 所示。

<a id="tab4-3"></a>

表 4-3 城市推荐模型指标与权重

| 指标 | 指标方向 | 权重 | 设置原因 |
|---|---|---:|---|
| 扣除生活成本和房租后的月结余 | 越高越好 | 0.38 | 直接反映毕业生实际剩余收入 |
| 平均税后月薪 | 越高越好 | 0.24 | 反映就业收入水平 |
| 市中心一居室租金 | 越低越好 | 0.18 | 房租是毕业生主要固定支出 |
| 单人月生活成本 | 越低越好 | 0.12 | 反映日常消费压力 |
| 就业热度代理指标 | 越高越好 | 0.08 | 辅助反映城市就业机会 |

如表 4-3 所示，本文将月结余设置为最高权重，原因是月结余已经综合考虑了工资、生活成本和租金，更接近毕业生真实生活状态。平均税后月薪权重排第二，主要体现收入水平。租金和生活成本为反向指标，数值越低表示压力越小。就业热度代理指标权重较低，因为它只能反映城市就业机会的大致吸引力，不能替代真实招聘平台岗位数量。

### 4.2.2 推荐分计算方法

由于不同指标单位不同，不能直接相加。例如，月薪和租金是“元/月”，就业热度代理指标是标准化后的相对值。因此，本文先对各指标进行 0-1 标准化，再按照权重进行加权求和。推荐分计算流程如图 4-2 所示。

<a id="fig4-2"></a>

![图4-2 城市推荐分计算流程图](D:/DataVisualization/大作业/zjh/figures/图4-2_城市推荐分计算流程图.png)

图 4-2 城市推荐分计算流程图

从图 4-2 可以看出，推荐分计算主要包括原始指标输入、指标标准化、权重设置、加权求和和推荐排序五个步骤。对于正向指标，例如月结余、平均薪资和就业热度，数值越大得分越高；对于反向指标，例如租金和生活成本，数值越小得分越高。

本文推荐分的基本计算思路如下：

```text
推荐分 =
0.38 × 标准化月结余
+ 0.24 × 标准化平均薪资
+ 0.18 × 反向标准化租金
+ 0.12 × 反向标准化生活成本
+ 0.08 × 标准化就业热度
```

这种计算方式比较简单，适合课程项目使用。它的优点是每个指标含义清晰，结果容易解释；不足是权重仍然带有一定主观性，不同用户如果偏好不同，推荐结果也可能不同。例如，有些学生更看重就业机会，可以提高就业热度权重；有些学生更看重低成本生活，可以提高租金和生活成本权重。

### 4.2.3 城市推荐结果解释

根据上述推荐分计算方法，本文对 20 个样本城市进行排序。推荐分排名前十的城市如表 4-4 所示。

|---:|---|---:|---:|---:|---:|
| 1 | 长沙 | 0.761 | 8562 | 1895 | 3072 |
| 2 | 成都 | 0.699 | 8389 | 2995 | 2204 |
| 3 | 南京 | 0.682 | 9259 | 3464 | 2317 |
| 4 | 青岛 | 0.656 | 7882 | 2350 | 2170 |
| 5 | 宁波 | 0.656 | 8189 | 2625 | 2183 |

| 7 | 深圳 | 0.653 | 11243 | 5312 | 1986 |

如表 4-4 所示，长沙推荐分最高，主要原因是其实际月结余较高，租金压力较小。成都、南京、青岛、宁波等城市也排在前列，这些城市并不一定是薪资最高的城市，但收入和成本之间相对更平衡。深圳虽然租金较高，但税后月薪和就业热度较高，因此推荐分仍然进入前十。

从推荐结果可以看出，推荐模型并不是简单地把工资最高的城市排在最前，而是综合考虑收入、成本和就业机会。这样的结果更符合本文研究主题，即城市选择不能只看工资，还要看生活成本、租金和实际月结余。

## 4.3 讨论环节：模型结果整合与洞察提炼

通过城市分型和推荐模型，可以对不同城市适合的人群进行进一步总结。模型结果不能直接替代个人决策，但可以帮助毕业生更有针对性地筛选城市。

### 4.3.1 不同城市类型的特征总结

从城市分型结果看，高薪高成本型城市主要包括北京、上海、深圳、广州等。这类城市就业机会多，行业平台较好，适合追求职业发展机会、薪资上限和大平台经验的毕业生。但是，这类城市租金和生活成本较高，如果刚毕业时薪资不够高，实际生活压力会比较大。

均衡发展型城市主要包括南京、成都、宁波、杭州、苏州等。这类城市的薪资水平和生活成本之间相对平衡，既有一定就业机会，也不会像一线城市那样承担过高租金压力。对于希望兼顾职业发展和生活质量的毕业生来说，这类城市具有较强参考价值。

低成本友好型城市主要包括长沙、青岛、天津、郑州、东莞等。这类城市的共同特点是租金相对较低，扣除生活成本和房租后的月结余表现较好。对于预算敏感、希望降低生活压力的毕业生来说，这类城市可能更适合起步阶段发展。

### 4.3.2 对毕业生就业城市选择的启示

从模型结果可以得到一个比较直接的启示：毕业生选择城市时不能只看平均工资。高薪城市往往伴随高房租和高消费，如果只关注收入，容易忽略真实生活压力。更合理的方式是同时比较税后月薪、生活成本、租金和月结余。

对于追求职业平台和高薪机会的学生，可以优先考虑北京、上海、深圳、杭州等城市，但需要提前评估房租支出和储蓄压力。对于希望生活压力较小、能够稳定积累资金的学生，可以关注长沙、成都、南京、青岛、宁波等城市。对于专业方向更偏制造、供应链、工程技术的学生，则可以结合苏州、东莞、佛山、南京等城市的产业结构进行选择。

此外，城市选择也不应该完全由模型决定。模型只能反映数据层面的比较，个人家庭情况、专业方向、职业规划、是否接受合租、是否在本地有人脉资源等因素，也会影响最终选择。因此，本文的推荐结果更适合作为初步筛选工具，而不是唯一决策依据。

### 4.3.3 模型结果的适用范围与局限

本文模型具有一定参考价值，但也存在局限。首先，生活成本和租金数据主要参考公开网页和用户贡献型平台，可能与实际租房市场存在差异。其次，就业结构数据来自公开报告和整理数据，不能代表实时全量招聘岗位。再次，推荐模型中的权重设置带有一定主观性，不同用户偏好不同，最合适的权重也会不同。

因此，本文模型更适合用于课程项目中的城市比较和可视化展示，而不是作为严格的就业决策模型。后续如果继续改进，可以接入更细粒度的真实招聘岗位数据、通勤成本、房价、行业增长率和时间序列数据，并允许用户自定义权重，使推荐结果更符合个人需求。

总体来看，本章通过城市分型和推荐分计算，将前文的描述性统计结果进一步转化为城市类型和推荐排序。结果表明，长沙、成都、南京、青岛、宁波等城市在收入成本匹配方面表现较好；北京、上海、深圳等城市虽然薪资和机会较强，但生活成本压力也更明显。对于毕业生而言，选择就业城市时应综合考虑薪资、生活成本、租金、岗位结构和个人发展目标。
