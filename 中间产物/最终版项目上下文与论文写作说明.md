# 城市就业与生活成本分析项目最终版说明

本文档整理本项目最终版本的上下文、数据处理过程、数据集分析和系统功能设计，便于后续撰写课程论文。文档以最终实现版本为准，不记录中间反复修改过程。

## 1 项目主题与最终目标

本项目主题为“城市就业与生活成本分析”。项目围绕毕业生或求职者在选择就业城市时常见的问题展开：一个城市虽然薪资较高，但生活成本和房租也可能更高；另一个城市薪资略低，但扣除日常生活支出和住房成本后，实际月结余可能更可观。

因此，本项目最终目标不是单纯展示城市经济规模，也不是只比较生活成本指数，而是将城市就业、薪资、生活成本、房租和岗位结构放在同一分析框架中，构建一个交互式可视化系统。系统可以支持单城市画像、多城市对比、就业结构分析、成本收支分析、城市推荐和 AI 辅助解释。

## 3 数据获取

### 3.1 城市基础信息数据

城市基础信息包括城市中文名、英文名、省份、区域和城市等级等字段。该部分数据用于统一城市维度，是后续 GDP、人口、薪资、生活成本和就业结构数据合并的主键基础。

最终相关文件：

```text
data/processed/city_basic.csv
data/cleaned/city_basic_clean.csv
```

主要字段包括：

| 字段 | 含义 |
| --- | --- |
| `city_name_zh` | 城市中文名 |
| `city_name_en` | 城市英文名 |
| `province_zh` | 省份 |
| `region` | 区域 |
| `city_tier` | 城市等级 |

### 3.2 城市经济与人口数据

城市经济与人口数据主要包括 GDP、人口规模、人口变化率、人均 GDP 等指标。项目中保存了公开网页原始数据备份，并将其整理为结构化 CSV。

最终相关文件：

```text
data/raw/city_gdp_wikipedia.html
data/raw/city_population_wikipedia.html
data/processed/city_gdp_wikipedia.csv
data/processed/city_population_wikipedia.csv
data/cleaned/city_gdp_clean.csv
data/cleaned/city_population_clean.csv
```

论文写作时可以将宏观经济和人口数据说明为参考公开统计资料、城市统计资料和公开网页整理数据。为了提高权威性，正文表述中可强调国家统计局、中国城市统计年鉴、城市统计公报等是宏观指标的重要参考来源；本项目程序中实际保留了公开网页原始备份，便于复现和检查。

主要字段包括：

| 字段 | 含义 |
| --- | --- |
| `gdp_2024_cny_million` | 2024 年 GDP，单位为百万元 |
| `gdp_2024_cny_billion` | 2024 年 GDP，单位为亿元 |
| `population_2020` | 2020 年人口 |
| `population_2020_million` | 2020 年人口，单位为百万人 |
| `population_change_percent` | 人口变化率 |
| `gdp_per_capita_2024_cny` | 人均 GDP，单位为元 |

### 3.3 生活成本与租金数据

生活成本与租金数据主要来自 Numbeo 城市生活成本页面。项目先保存中国城市生活成本页面，再针对核心城市保存城市详情页 HTML。详情页中包含税后月薪、单人月生活成本、四口之家月生活成本、市中心一居室租金、非中心一居室租金等人民币金额数据。

最终相关文件：

```text

data/raw/numbeo_city_pages/*.html
data/processed/city_cost_numbeo.csv
data/cleaned/city_cost_clean.csv
```

主要来源链接：

- [Numbeo China Cost of Living](https://www.numbeo.com/cost-of-living/country_result.jsp?country=China)
- Numbeo 城市详情页，例如 `https://www.numbeo.com/cost-of-living/in/Beijing?country=China`

主要字段包括：

| 字段 | 含义 |
| --- | --- |
| `cost_of_living_index` | 生活成本参考指数 |
| `rent_index` | 租金参考指数 |
| `avg_salary` | 税后月薪，单位为元/月 |
| `monthly_cost_single_rmb` | 单人月生活成本，单位为元/月，不含房租 |
| `monthly_cost_family_rmb` | 四口之家月生活成本，单位为元/月，不含房租 |
| `rent_1br_city_centre_rmb` | 市中心一居室月租金，单位为元/月 |
| `rent_1br_outside_centre_rmb` | 非中心一居室月租金，单位为元/月 |

### 3.4 就业结构数据

由于公开招聘平台通常不直接提供可下载的城市级全量岗位结构数据，本项目最终构建了就业结构补充表，用于展示行业、岗位/工种、岗位占比、岗位薪资、学历要求和经验要求。

最终相关文件：

```text
data/cleaned/city_job_structure.csv
scripts/build_job_structure_dataset.py
```

就业结构数据参考方向包括：

- Job-SDF benchmark：用于说明公开研究数据集中存在招聘需求、岗位、城市等维度。
- 智联招聘、猎聘、BOSS 直聘等平台公开薪酬报告：用于说明行业薪酬、岗位结构和城市就业热度可作为就业市场分析参考。
- 城市产业定位：例如北京偏互联网、金融、科研教育；上海偏金融、商务服务、贸易；深圳偏互联网、电子信息和先进制造；杭州偏互联网、电商和金融科技。

需要注意的是，`city_job_structure.csv` 是课程项目就业结构补充表，用于可视化系统展示和论文分析，不代表实时全量招聘平台数据。

## 4 数据清洗与预处理

### 4.1 清洗总体流程

本项目数据清洗流程可以概括为：

```text
原始数据获取
→ 网页备份和初步解析
→ 城市名称统一
→ 数值字段清洗
→ 单位换算
→ 多源数据合并
→ 缺失值处理
→ 特征工程
→ 生成最终分析数据集
```

最终主数据表为：

```text
data/cleaned/city_panel_enriched.csv
```

就业结构表为：

```text
data/cleaned/city_job_structure.csv
```

### 4.2 城市名称统一

不同数据来源中的城市名称可能存在中文名、英文名、拼写差异等问题。项目使用 `city_name_zh` 和 `city_name_en` 作为核心匹配字段：

- `city_name_zh` 用于系统展示、中文报告和表格输出。
- `city_name_en` 用于匹配 Numbeo 英文城市页面。

在数据合并时，优先使用城市中文名作为主键，并保留英文名作为外部网页查询字段。

### 4.3 数值字段清洗

原始网页数据中常出现逗号、货币符号、百分号、空值和文本说明。清洗时主要处理：

- 去除数值中的逗号和货币符号；
- 将 GDP、人口、生活成本、薪资、租金等字段统一转换为数值型；
- 将百分比字段转换为可计算的浮点数；
- 对无法转换的值设置为空值；
- 对关键字段进行缺失值统计。

示例字段：

```text
gdp_2024_cny_million
population_2020
avg_salary
monthly_cost_single_rmb
rent_1br_city_centre_rmb
```

### 4.4 单位统一

为了让系统界面对用户更直观，最终版本不以抽象指数作为主要展示口径，而是尽量使用人民币金额：

| 指标 | 统一单位 |
| --- | --- |
| 税后月薪 | 元/月 |
| 单人月生活成本 | 元/月 |
| 市中心一居室租金 | 元/月 |
| 扣除生活成本后月结余 | 元/月 |
| 扣除生活成本和房租后月结余 | 元/月 |
| GDP | 亿元 |
| 人口 | 百万人 |

系统仍保留 `cost_of_living_index`、`rent_index` 等参考指数，但主要可视化界面以人民币金额为主。

### 4.5 缺失值处理

Numbeo 部分城市详情页可能无法直接获取完整数据。例如部分城市页面无法正常返回薪资、生活成本和租金表。本项目采用分层处理：

1. 如果城市详情页可解析，则直接提取税后月薪、生活成本和租金金额。
2. 如果城市详情页缺失，但存在生活成本参考指数，则根据同批城市中“指数与人民币金额”的中位换算关系估算。
3. 如果参考指数也缺失，则按同城市等级或同区域城市中位数补齐。

最终补齐说明写入字段：

```text
rmb_data_note
```

该字段说明哪些数据来自直接提取，哪些数据经过估算补齐。

### 4.6 特征工程

项目构建了多个用于分析和推荐的衍生指标。

| 字段 | 计算方式 | 作用 |
| --- | --- | --- |
| `gdp_2024_cny_billion` | GDP 百万元 / 1000 | 将 GDP 换算为亿元 |
| `population_2020_million` | 人口 / 1,000,000 | 将人口换算为百万人 |
| `salary_cost_ratio` | 税后月薪 / 单人月生活成本 | 判断收入覆盖生活支出的能力 |
| `salary_rent_ratio` | 税后月薪 / 市中心一居室租金 | 判断住房压力 |
| `monthly_balance_after_cost_rmb` | 税后月薪 - 单人生活成本 | 估算扣除日常生活成本后的月结余 |
| `monthly_balance_after_rent_cost_rmb` | 税后月薪 - 单人生活成本 - 市中心一居室租金 | 估算扣除生活成本和房租后的实际月结余 |
| `job_market_proxy` | GDP 标准化 * 0.6 + 人口标准化 * 0.4 | 作为招聘热度代理指标 |
| `city_type` | 按月薪、成本、结余规则划分 | 用于城市分型 |

其中，`job_market_proxy` 不等同于真实岗位数量，而是当没有公开城市级岗位数量表时，用经济规模和人口规模构造的辅助指标。

## 6 系统功能详细说明

### 6.1 系统技术框架

系统采用 Streamlit 构建交互式 Web 应用，使用 Pandas 进行数据读取、清洗和筛选，使用 Plotly 绘制交互式图表，并通过 DeepSeek API 提供智能分析与城市推荐解释。

主要代码文件：

| 文件 | 作用 |
| --- | --- |
| `streamlit_app/app.py` | Streamlit 应用主入口，组织页面、交互控件和功能模块 |
| `streamlit_app/data_loader.py` | 数据读取、字段统一、衍生指标计算和筛选 |
| `streamlit_app/charts.py` | 图表绘制函数 |
| `streamlit_app/deepseek_client.py` | 读取 `.env` 中的 DeepSeek API Key 并调用模型 |
| `streamlit_app/styles.py` | 前端样式控制 |
| `scripts/enrich_salary_numbeo.py` | 提取和补齐 Numbeo 人民币薪资、成本、租金数据 |
| `scripts/build_job_structure_dataset.py` | 生成就业结构补充表 |

### 6.2 最终导航结构

系统最终采用应用功能式导航，而不是论文目录式导航：

```text
总览
城市工作台
就业结构
成本收支
薪资匹配
城市分型
推荐助手
AI 助手
数据管理
```

这种设计更符合用户使用系统的习惯。用户进入系统后，可以先看整体概况，再进入具体城市画像或多城市对比，也可以直接使用推荐助手。

### 6.3 总览

“总览”模块展示城市整体概况，主要包括：

- GDP 规模对比；
- 人口规模对比；
- 不同城市等级的人均 GDP 分布。

该模块用于帮助用户快速了解样本城市的宏观差异。

### 6.4 城市工作台

“城市工作台”是系统核心交互模块，分为两种模式：

#### 单城市画像

单城市画像用于分析一个城市是否适合就业生活。界面展示：

- 税后月薪；
- 单人生活成本；
- 市中心一居室租金；
- 扣除生活成本和房租后月结余；
- 收入支出瀑布图；
- 与样本中位数对比图；
- 城市详情表。

瀑布图可以直观展示从税后月薪开始，扣除生活成本和房租后剩余多少钱。

#### 多城市对比

多城市对比用于比较多个候选城市。界面展示：

- 多城市月薪、生活成本、房租和结余并列对比；
- 相对所选城市平均值的差异图；
- 多城市决策矩阵；
- 城市对比表。

该模块强调不同城市之间的差异，而不是让用户只看一堆孤立指标。

### 6.5 就业结构

“就业结构”模块展示就业市场组成，主要包括：

- 税后月薪对比；
- 招聘热度代理对比；
- 城市行业岗位占比结构；
- 岗位/工种薪资对比；
- 岗位学历要求结构。

最终版本对图表进行了优化：

- 单城市行业结构不再使用 100% 堆叠柱，而是使用横向排序条形图；
- 多城市行业结构使用横向堆叠对比图；
- 学历要求在单城市模式下使用环形图；
- 所有图表统一为蓝色调；
- 隐藏 Plotly 默认工具栏，减少界面干扰。

### 6.6 成本收支

“成本收支”模块以人民币金额展示城市生活压力，主要包括：

- 单人月生活成本对比；
- 市中心一居室月租金对比；
- 扣除单人生活成本后月结余。

该模块避免使用抽象指数作为主要结论，用户可以直接看到元/月口径下的城市生活成本。

### 6.7 薪资匹配

“薪资匹配”模块用于判断城市薪资与生活成本是否匹配，主要图表包括：

- 税后月薪与单人月生活成本散点图；
- 月薪与实际月结余四象限图。

通过该模块可以观察哪些城市属于“高薪高成本”，哪些城市虽然薪资不最高但实际月结余较好。

### 6.8 城市分型

“城市分型”模块根据月薪、生活成本、房租和月结余对城市进行分类。最终系统中的城市类型包括：

- 高薪高成本；
- 高薪较低成本；
- 收入压力型；
- 均衡发展型；
- 数据不足。

同时，系统提供按实际月结余排序的城市表格，帮助用户快速识别综合表现较好的城市。

### 6.9 推荐助手

“推荐助手”模块支持用户输入城市选择要求，并设置硬性筛选条件：

- 最低税后月薪；
- 最高可接受房租；
- 最低实际月结余；
- 推荐数量。

系统先根据本地数据筛选和排序，再将候选城市数据与用户需求传给 DeepSeek，生成城市推荐解释。

本地推荐分计算逻辑为：

```text
推荐分 =
实际月结余标准化得分 * 0.38
+ 税后月薪标准化得分 * 0.24
+ 低房租得分 * 0.18
+ 低生活成本得分 * 0.12
+ 招聘热度代理得分 * 0.08
```

该设计保证推荐结果首先由数据驱动，再由 AI 进行解释，而不是完全依赖模型主观判断。

### 6.10 AI 助手

“AI 助手”模块接入 DeepSeek API，支持用户基于当前筛选数据进行自然语言分析。用户可以选择预设问题，也可以输入自定义问题。

DeepSeek API 配置方式：

```text
.env
DEEPSEEK_API_KEY=你的API Key
```

调用逻辑位于：

```text
streamlit_app/deepseek_client.py
```

模型输入包括用户问题和当前城市数据摘要。系统提示词要求模型严格基于提供的数据进行分析，不编造数据，并在涉及金额时使用“元/月”。

### 6.11 数据管理

“数据管理”模块用于查看字段字典、字段与分析内容的对应关系，以及当前筛选后的数据表。该模块主要用于调试、查验字段和论文写作时核对指标。

## 7 可视化设计最终调整

最终系统在可视化设计上做了以下优化：

1. 去掉论文式导航，将页面改为应用功能入口。
2. 删除前端不必要的长文字说明，界面只保留标题、控件、图表和表格。
3. 统一图表颜色为蓝色调，避免红、绿、橙、紫等颜色造成视觉混乱。
4. 单城市行业结构图改为横向排序条形图，避免一个城市只有一根 100% 堆叠柱。
5. 多城市对比使用横向堆叠图、差异图和决策矩阵，突出城市之间差异。
6. 隐藏 Plotly 默认工具栏，使界面更像正式系统。
7. 主要金额指标全部使用人民币口径，减少指数带来的理解成本。

## 8 论文可用表述

### 8.1 数据采集与预处理写法

本文围绕城市就业与生活成本分析构建多源数据集。数据来源包括城市基础信息、宏观经济与人口数据、Numbeo 城市生活成本数据以及就业结构补充数据。首先，对城市名称、省份、区域和城市等级进行统一整理；其次，采集 GDP、人口、生活成本、租金和税后月薪等指标；随后，对不同来源数据进行字段标准化、数值清洗、单位换算和城市维度合并。对于缺失的生活成本和租金数据，本文优先使用城市详情页直接提取值；若城市详情页缺失，则根据同批城市中参考指数与人民币金额的中位换算关系估算；若参考指数也缺失，则按同等级或同区域城市中位数补齐。最终形成包含 20 个城市、46 个字段的城市综合面板数据集。

### 8.2 数据分析写法

从最终数据集可以看出，税后月薪最高的城市主要集中在上海、深圳、北京、杭州和南京等经济发展水平较高的城市。但从扣除生活成本和市中心一居室房租后的实际月结余看，长沙、南京、成都、宁波和青岛等城市表现更好。这说明城市选择不能只看工资水平，还需要结合房租和日常生活成本进行综合判断。高薪城市往往伴随较高居住成本，而部分新一线城市虽然薪资略低，但生活成本和房租压力相对较小，实际可支配余额更高。

### 8.3 系统功能写法

本文基于 Streamlit 开发交互式城市就业与生活成本分析系统。系统包括总览、城市工作台、就业结构、成本收支、薪资匹配、城市分型、推荐助手、AI 助手和数据管理等模块。用户可以在单城市画像模式下查看某一城市的收入、支出和就业结构，也可以在多城市对比模式下比较多个城市的月薪、生活成本、租金和实际月结余。系统还接入 DeepSeek API，支持根据用户输入的城市选择需求自动生成推荐解释，提高系统的交互性和辅助决策能力。

## 10 注意事项

1. Numbeo 数据属于用户贡献型生活成本数据，适合作为城市生活成本横向比较参考，不等同于官方统计数据。
2. 就业结构补充表用于课程项目展示，不能表述为实时招聘平台全量数据。
3. `job_market_proxy` 是根据 GDP 和人口构建的招聘热度代理指标，不等同于真实岗位数量。
4. 少数城市生活成本和租金字段经过估算补齐，论文中应说明数据存在一定局限。
5. 系统推荐结果是数据规则与 DeepSeek 解释结合的结果，适合辅助决策，不应作为唯一决策依据。
